Why GoldenGate Sits Inside AVDF

REDO (transaction) logs record the before and after image of every DML/DDL change made to an Oracle Database, so the instance can recover after a crash. Oracle Audit Vault and Database Firewall reuses this same stream — via GoldenGate's Integrated Extract — as its TRANSACTION LOG audit trail, instead of relying only on native database auditing. This gives AVDF visibility into the actual column-level before/after values changed by a transaction.

End-to-end flow

📁 Source Oracle DB
REDO / archive logs. Supplemental logging + force logging enabled. 12.2.0.1+ required for direct Integrated Extract.
→
⚙ Integrated Extract
Runs on source DB (12.2.0.1+) or a Downstream Mining DB (pre-12.2.0.1). Captures DML + DDL for configured TABLE / DDL INCLUDE objects.
→
📜 XML Trail Files
OUTPUTFORMAT XML _AUDIT_VAULT. Trail rotates at configured Trail Size (MB). Written to a directory, read directly — not consumed as a normal GG trail.
→
📡 AVDF Transaction Log Collector
Audit Vault Agent reads the XML trail directory. Checkpoints progress into an .atc (Audit Trail Cleanup) file, in UTC.
→
🔐 Audit Vault Server
Records land in event_log. DDL → All Activity report. DML → Data Modification Before-After Values report (with key columns since 20.10).

⚙ Integrated Extract

The only supported capture mode for 12.2.0.1+ — runs inside the database's logmining server, directly against the source. One Integrated Extract per database is the recommended baseline.

🔁 Downstream Mining

Required for source databases older than 12.2.0.1: the source ships REDO to a 12.2.0.1+ mining database, and Integrated Extract runs there instead of on the source.

🔑 Key Columns

Default to the table's primary key. GoldenGate's KEYCOLS option lets you override this when no PK exists, or a different column set is preferred for the report.

🔍 GETBEFORECOLS

Needed when key columns aren't already present in the before-image, so update/delete key values still render correctly in the AVDF report (available from AVDF 20.10).

Terminology convention used throughout this simulator: Integrated Extract (not Classic Extract), Downstream Mining database, and OUTPUTFORMAT XML _AUDIT_VAULT — matching the AVDF 20 Administrator's Guide exactly.
